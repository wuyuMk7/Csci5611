Description
  • The video contains an agent starting from the position (-9, 9) and heading the target (9, 9).
  • The obstacle is located at (0, 0).
  • The program sampled 200 points and connected two points if their distance is not greater than 5.0.
  • Black lines represent the edge of the sample graph.
  • Red line represents the predicted shortest path from the starting point to the target.
  • The agent is also painted red.

Difficult parts
  • In my program, there is a parameter controlling the maximum searching radius while creating edges between sample points. If the distance of two points is greater than the radius, then the program will not add an edge between them. With this setting, the path can be better but a problem is that the sample points may be divided into several separated parts if the least distance between two parts is still greater than the threshold radius. In this case, the program cannot find a path from the starting point to the target. For the check-in, I tuned this parameter and the number of sample points to avoid this issue. But I think a better solution may be to do a double check over the graph after creating edges and to add links among the separated parts to connect all of them.
